Scheduler and Pollers

How’s and Why’s
  • How to trigger the integration batches
  • How to process the on FTP server
  • Can we schedule a webservice callout / or API invocation.
  • Should we leverage timers / or event based triggers.
Perspectives
  • I am an Architect, so we should understand a few things:
    • Processing Duration of a Batch
    • Secure Connection with FTP, File Sizing
    • Data Parsing Requirements
    • Scheduler for direct WS Callout / API invocation is a dangerous thing, and we should propose a trigger components to mediate the call
  • I am an Developer, so we have to make it work.
    • What components are available to make a connection to File Server.
    • How I am going to read a record, validate it and transform into a desired Mime / Type.
  • We are a team, so we have to scale it.
    • We can’t read the same file again.
    • Record processing can fail, and should the failed records be picked in next run.
    • We have to keep hooks for Notification and have coverage for system errors like batch / file hasn’t been picked for an hour / a day.
Some Understanding
  • Inputs from EA group should be the basis for the design approach and development tasks.
  • Generally, we should strive for separation of record level from File / Batch level.
  • Irrespectively, keep repeating the tasks and we will see the record in the target system.

Learn from Open Source

We all know about Quartz, but generally what we may see in real integration environments are components developed on open source Quartz framework. However we can use quartz-scheduler maven dependencies to better understand and use the components available with product and platforms. 

                     
                    // maven dependency
                    <dependency>
                    <groupId>org.quartz-scheduler</groupId>
                    <artifactId>quartz</artifactId>
                    <version>2.1.5</version>
                    </dependency>
                    import org.quartz.Job;
                    // Our Job class has to implement Quartz Job Interface, and implementation provided in the execute method will be run based on schedule frequency. 
                    public void execute(JobExecutionContext context)
                        throws JobExecutionException {
                        System.out.println("Hello Quartz!");
                    }
                    // In the Application we use the Job Builder and Trigger Builder to create job and trigger instance. We get the scheduler instance from factory class and we pass the job and trigger instance to scheduleJob method of scheduler instance.
