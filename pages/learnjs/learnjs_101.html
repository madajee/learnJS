Building Objects

  • In Javascript, we work with primitive types, arrays, objects and functions, but object is the most important data type as it is the container for properties & methods.
  • Simplistically, we can create an object using an object literal. 
                     
                var car = {make: “Nissan”, model: “Altima”}
  • Object Constructor is another way of creating an object. 
                     
                const car = new Object();
                car.make = ‘nissan’;
  • Function Constructor is the recommended way when we have to instantiate several objects with the same properties and methods. It is also inline with OOPs way of creating objects from class blueprints. 
                     
                function car (make, model) {
                    this.make = make;
                    this.model =  model;
                }
                let vehicleA = new car(‘toyota’, ‘camry’);
                let vehicleB = new car(‘honda’, ‘civic’);
    When we create objects with a Function Constructor, the __proto__ property of the new object will be an empty object of type car.
  • Creating Objects with Prototypes
    JS Engine provides Object.create(obj) to create objects from prototype object and it does it by assigning the prototype of new object to base object. This uses pure prototypical inheritance to create objects. 
                     
                    var car = {make: “Nissan”, model: “Altima”, displayInfo: function() {console.log('Car with Make: ' + this.make + ' Model: ' + this.model}};
                    var camry = Object.create(car);
                    console.log(‘Prototype of new object will be car: ‘ + camry.__proto__);

Prototypical Inheritance

  • Inheritance as a concept is a child object having access to the properties and methods of a parent object.
  • Javascript implements the concept of inheritance with prototypes and is referred to as prototypical inheritance.
  • Rather than using a blueprint like class, access modifiers for encapsulation, everything in Javascript is an object, and all objects have __proto__ property which has a reference to another object.
  • Objects have access to its properties and methods but it also has access to properties and methods in a prototype chain.
  • One of the ways we have discussed earlier is to use function constructor to create objects. When we use ‘new’ keyword, JS Engine creates an empty object, references the function constructor object as the prototype of the new object. All functions have a prototype object, so the new object will have access to all methods that are on the prototype of the function constructor.

Object Oriented Concepts in Javascript

Classes are generally the blueprint, and we create objects as instances of classes in other programming languages. Javascript classes, introduced in ECMA2015, is primarily a syntactical sugar over Javascript’s prototype based inheritance. 

             
            class carES6 {
                constructor(make, model) {
                    this.make = make;
                    this.model =  model;
                }
                displayCar() {
                    return ('Car with Make: ' + this.make + ' Model: ' + this.model);
                }
            }
                let vehicleES6 = new carES6('toyota', 'camry');
                console.log(vehicleES6.displayCar());

Encapsulation One of key OOP concepts is that data can only be accessed with method, and in other programming languages we have access modifiers like public, protected and private to define access on class properties. We can implement the same in Javascript using the let keyword to define the properties of the constructor function and methods within function scope can expose these properties. 

             
            // Encapsulation
            function PersonEnc(firstname, lastname) {
                let fname= firstname;
                let lname= lastname;
                this.getFullName = function() {
                return fname + ' ' + lname;
                }
            }
            var johnenc = new PersonEnc('John', 'Doe');
            console.log('fname and lname can only bbe accessed in getFullName: ' +johnenc.fname);
            console.log(johnenc.getFullName());
