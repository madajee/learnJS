AWS 101

Understanding Identity and Access Management (IAM)

  • At the highest level is an organization which usually has a master account
  • Organization can have multiple organization units and each org unit can have one or more accounts.
  • IAM Identities belong to an account and should be provided permissions to access resources in an account.
  • When you a small org, you create limited IAM users, couple of groups to manage your users and create policies to provide permission to the users.
  • Roles in AWS are short term priviledges that are assigned to IAM identities to control access to resources. Roles in AWS platform is also used for providing access to resources from application and AWS services
  • After the successful authentication and acquiring grants based on the role, appropriate authorization is controlled with the application of AWS policies
  • Either when the org grows or multiple projects teams start using AWS, you will have scenarios where you will shape up your AWS Organization.

How Authorization Works

AWS is designed with the principle of least priviledges which mean you need permission for everything.

  • if any policies denies -> Access Denied
  • If some policy allows -> Allow
  • Else Access Denied

Examples

  • Read Data from DynamoDB
    •                      
                   {
                       "Effect": "Allow",
                       "Action": [
                        "dynamodb:GetItem",
                        "dynamodb:PutItem" 
                       ],
                       "Resource" : [
                        "arn:aws:dynamodb:us-east-2:resourceidentifier:table/tablename"
                       ]
                   }
  • Launch Specific EC2 Instance Types
    •                      
                   {
                       "Effect": "Allow",
                       "Action": "ec2:RunInstance",
                       "Resource" : [
                        "arn:aws:ec2:*:*:subnet/*",
                        "arn:aws:ec2:*:*:security-group/*",
                        "arn:aws:ec2:*:*:key-pair/*",
                        "arn:aws:ec2:*:*:volume/*",
                        "arn:aws:ec2:*:*:image/*",
                        "arn:aws:ec2:*:*:network-interface/*"
                       ]
                   }
  • Prevent unencrypted writes to S3
    •                      
                   {
                       "Effect": "DENY",
                       "Action": "s3:PutObject",
                       "Resource" : "arn:aws:s3:::example-bucket/*",
                       "Condition" : [
                        "StringNotEqualIfExists": {
                            "s3:x-amz-server-side-encryption-aws-kms-key-id": [
                                "arn:aws:kms:us-east-2:11122223333:key/12345678-1234-1234-1234-12345678abcd"
                            ]
                        }
                       ]
                   },
                   {
                    "Effect": "Allow",
                    "Action": ["s3:PutObject","s3:GetObject"],
                    "Resource" : "arn:aws:s3:::example-bucket/*"
                   }
